# ArcReady v0.3.0 Launch Package

## One-line positioning

ArcReady is an open-source Arc-specific static CI quality gate and integration-pattern validator for wallets, bridge flows, App Kit integrations, and dApps.

## Short product description

ArcReady helps developers catch common Arc integration mistakes before release. It runs locally or in CI, checks 18 active static rules across wallet, bridge, and App Kit presets, and generates terminal, JSON, Markdown, or HTML reports.

## Longer product description

ArcReady is a developer-side early warning layer for Arc-specific integration patterns. It looks for common assumptions that can slip into Arc projects, such as Ethereum-style gas labels, multi-confirmation waits, incorrect CCTP domain values, wrapped USDC routing assumptions, relayer gas-token mistakes, and App Kit configuration gaps.

The project is designed to be lightweight and CI-friendly. Developers can run it with `npx`, add it to GitHub Actions with `tanka420/arcready@v0.3.0`, or use it locally before opening a pull request. ArcReady remains a static quality gate: it does not perform live RPC checks, Circle API checks, runtime App Kit validation, bridge execution simulation, or on-chain transaction simulation.

## Demo video script

Arc integrations can fail because of small configuration and assumption mistakes: Ethereum-style gas labels, too many confirmation waits, wrong CCTP domains, wrapped USDC assumptions, or missing App Kit guardrails.

ArcReady is an open-source static CI quality gate for those patterns. It checks wallet, bridge, and App Kit integrations before release and gives developers actionable findings.

First, initialize a config:

```powershell
npx --yes arcready@0.3.0 init
```

Then run a scan:

```powershell
npx --yes arcready@0.3.0 scan
```

ArcReady reports the project score, status, summary counts, affected files, and suggested fixes. It can also write Markdown or HTML reports for CI artifacts:

```powershell
npx --yes arcready@0.3.0 scan --format markdown --output arcready-report.md
npx --yes arcready@0.3.0 scan --format html --output arcready-report.html
```

For GitHub Actions, add:

```yaml
- uses: tanka420/arcready@v0.3.0
```

ArcReady is a static quality gate. It does not simulate bridge execution, call Circle APIs, run live Arc RPC checks, or validate App Kit at runtime.

Try it with `npx`, add it to CI, and open an issue with Arc integration patterns you want checked.

## What ArcReady checks

Wallet preset:

- Arc chain metadata
- native USDC display
- ETH gas labeling assumptions
- one-confirmation finality assumptions
- unsupported PREVRANDAO assumptions
- blob transaction assumptions

Bridge preset:

- one-confirmation bridge assumptions
- CCTP domain `26`
- canonical USDC vs wrapped USDC assumptions
- relayer gas token assumptions
- retryable CCTP attestation `404` handling
- avoiding PREVRANDAO relay selection assumptions

App Kit preset:

- `Arc_Testnet` / Arc chain identifier usage
- capability guardrails
- explicit RPC configuration
- user-controlled wallet delegate handling
- user-facing fee explanation
- bridge minimum amount copy

## What ArcReady does not do

ArcReady does not perform:

- live Arc RPC checks
- Circle API checks
- on-chain transaction simulation
- contract deployment validation
- bridge runtime simulation
- real App Kit runtime validation
- SaaS/dashboard monitoring

ArcReady findings are static developer-side release checks. They are not runtime guarantees or official compatibility certification.
