## Launch positioning

ArcReady is an open-source Arc-specific static CI quality gate and integration-pattern validator for wallets, bridge flows, App Kit integrations, and dApps.

## LinkedIn post - primary version

I released ArcReady v0.3.0, an open-source Arc-specific static CI quality gate and integration-pattern validator.

ArcReady helps developers catch common Arc integration mistakes before release across:

- wallets
- bridge flows
- App Kit integrations
- dApps

It includes 18 active static rules across wallet, bridge, and App Kit presets. It checks for integration patterns such as Arc chain metadata issues, native USDC display mistakes, ETH gas labeling assumptions, one-confirmation finality assumptions, CCTP domain `26`, canonical USDC routing, relayer gas-token assumptions, App Kit chain identifier usage, capability guardrails, explicit RPC configuration, Unified Balance UX copy, and bridge minimum amount guardrails.

You can run it locally:

```powershell
npx --yes arcready@0.3.0 scan
```

Or add it to GitHub Actions:

```yaml
- uses: tanka420/arcready@v0.3.0
```

ArcReady generates terminal, JSON, Markdown, and HTML reports, so it works both as a local developer check and a CI artifact producer.

The v0.3.0 release focuses on rule quality hardening: better false-positive handling, cross-preset regression coverage, and clearer finding messages and suggestions.

Scope note: ArcReady is a static quality gate, not a runtime simulator or official Circle/Arc validator. It does not perform live Arc RPC checks, Circle API checks, on-chain transaction simulation, contract deployment validation, bridge runtime simulation, real App Kit runtime validation, or SaaS/dashboard monitoring.

ArcReady is an independent open-source developer tool and is not an official Circle or Arc product.
