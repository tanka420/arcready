# ArcReady v0.3.0 Demo Recording Guide

## Demo objective

Record a practical 60-90 second demo that shows ArcReady as an open-source Arc-specific static CI quality gate and integration-pattern validator.

The demo should make three points clear:

- Arc integrations can break from small config and assumption mistakes.
- ArcReady catches common static integration patterns before release.
- ArcReady is easy to run locally or in CI with `arcready@0.3.0` and `tanka420/arcready@v0.3.0`.

## Recommended video length

Target 60-90 seconds.

Suggested pacing:

- 10 seconds: problem
- 15 seconds: what ArcReady is
- 25 seconds: local CLI demo
- 15 seconds: report and GitHub Action demo
- 10 seconds: scope and CTA

## Recording setup

- Use a clean terminal with readable font size.
- Open the ArcReady README in the browser or editor.
- Prepare a small fixture or sample project with an Arc-specific issue.
- Keep browser/editor tabs minimal: README, terminal, generated report, GitHub Action snippet.
- Avoid showing secrets, personal tokens, or private repo data.

## Terminal preparation

Start from a project folder that can safely receive an `arcready.config.json` file.

Before recording:

- clear the terminal
- make sure Node.js 22+ is available
- make sure internet access is available for `npx`
- optionally remove any old `arcready-report.md` or `arcready-report.html`

## Scene-by-scene script

1. Problem: "Arc integrations can fail because of small configuration and assumption mistakes."
2. Product: "ArcReady is an open-source Arc-specific static CI quality gate for wallet, bridge, and App Kit integration patterns."
3. Local demo: Run init and scan.
4. Reports: Generate Markdown or HTML output.
5. CI demo: Show GitHub Action usage with `tanka420/arcready@v0.3.0`.
6. Scope: Clarify static checks only.
7. CTA: Try it with `npx` or add it to CI.

## Exact terminal commands

Required demo commands:

```powershell
npx --yes arcready@0.3.0 init
npx --yes arcready@0.3.0 scan
npx --yes arcready@0.3.0 scan --format markdown --output arcready-report.md
npx --yes arcready@0.3.0 scan --format html --output arcready-report.html
```

GitHub Action usage:

```yaml
- uses: tanka420/arcready@v0.3.0
```

Current CLI note for live recording:

The current CLI writes files with `--out`. If recording directly against the current CLI, use:

```powershell
npx --yes arcready@0.3.0 scan --format markdown --out arcready-report.md
npx --yes arcready@0.3.0 scan --format html --out arcready-report.html
```

## Voiceover script

"Arc integrations can fail because of small assumption mistakes: wrong gas labels, too many confirmation waits, CCTP domain mismatches, wrapped USDC assumptions, or missing App Kit guardrails.

ArcReady is an open-source Arc-specific static CI quality gate. It checks wallet, bridge, and App Kit integration patterns before release.

First I initialize a config with `npx --yes arcready@0.3.0 init`. Then I run a scan with `npx --yes arcready@0.3.0 scan`.

ArcReady reports a score, status, findings, affected files, and suggested fixes. It can also generate Markdown, JSON, or HTML reports for CI artifacts.

For GitHub Actions, add `tanka420/arcready@v0.3.0`.

ArcReady is static analysis. It does not run live Arc RPC checks, call Circle APIs, simulate bridge execution, or validate App Kit at runtime.

Try it with `npx`, add it to CI, and open an issue with Arc integration patterns you want checked."

## GitHub Action segment

Show this minimal snippet:

```yaml
jobs:
  arcready:
    runs-on: ubuntu-latest
    permissions:
      contents: read

    steps:
      - uses: actions/checkout@v4

      - uses: tanka420/arcready@v0.3.0
        with:
          fail-on: critical
```

Voiceover:

"The GitHub Action runs the published npm CLI and defaults to `arcready@0.3.0`. It can upload JSON, Markdown, and HTML reports as CI artifacts."

## Scope disclaimer

ArcReady is a static developer-side release check. It does not perform:

- live Arc RPC checks
- Circle API checks
- on-chain transaction simulation
- contract deployment validation
- bridge runtime simulation
- real App Kit runtime validation
- SaaS/dashboard monitoring

ArcReady is an independent open-source project. It is not an official Circle or Arc product.
